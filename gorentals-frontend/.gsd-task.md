# UX Blueprint Requirements

## General Token Rules
Apply Emerald Equinox (Deep Teal) across ALL pages below.
- Primary: var(--primary) or teal.
- Remove all orange styling globally (especially from Help page).

## Core UI Context Hiding
- In `Navbar.tsx` and `Footer.tsx` logic:
  - Add `const pathname = usePathname();`
  - If `['/login', '/signup', '/forgot-password'].includes(pathname)`, return `null`.

## Footer Links Cleanup
- Complete removal of `/categories` route block.
- Update routes exactly:
  - Discover: Browse all items (/search), Owner stores (/stores)
  - Earn: List your item (/signup?role=OWNER)
  - Support: Help Center (/help), Terms (/terms), Privacy (/privacy)
- Remove 'Owner protection' and 'How it works' labels.

## Auth Overhaul
- `login/page.tsx`: Strip "Are you an admin?" escape route text/links completely to prevent admin leakage.
- `forgot-password/page.tsx`: Create a full-class styled, Teal-branded recovery page instead of a stub. Include email collection, robust validation, and success state toggle.

## Stores Page
- Rename header to "Owner Stores" instead of Archive.
- Remove physical shops from placeholder.
- Insert standard Teal EmptyState logic (Loading, No Stores, Filter mismatch).

## Help Page
- Purge orange. Replace with Teal.
- Provide a robust "Contact Support" embedded sub-form block.
